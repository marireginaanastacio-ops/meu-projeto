## Story
**As a** usuário,
**I want** configurar critérios de busca e disparar a coleta de prospects via Apify,
**so that** minha base de leads seja populada automaticamente sem trabalho manual.

## Acceptance Criteria
1. Endpoint `POST /api/prospects/search` aceita body: `{ platform: "instagram"|"linkedin", keywords: string, limit?: number }`
2. Apify Actor correto é acionado conforme plataforma: `apify/instagram-profile-scraper` (Instagram) ou `proxycurl/linkedin-profile-scraper` (LinkedIn)
3. Resultados são normalizados pelo `lead-transformer.js` para o schema padrão de Lead com campos: `id`, `nome`, `username`, `plataforma`, `bio`, `url`, `foto_url`, `seguidores`, `localizacao`, `keywords_busca`, `status: "Novo"`, `data_captura`, `updated_at`
4. Leads são persistidos em `data/leads.json` sem duplicatas (deduplicação por `url`)
5. Endpoint retorna `{ success: true, data: { added: N, duplicates: M, total: T } }`
6. Em caso de erro da API Apify, retorna `{ success: false, error: "mensagem", code: "APIFY_ERROR" }` sem corromper `leads.json`
7. `JsonStore` implementado em `server/src/utils/json-store.js` com cache em memória e escrita atômica sequencial

### JsonStore Pattern (Source: architecture/prospeccao-automatizada-architecture.md#5.1)

```javascript
class JsonStore {
  constructor(filePath) {
    this.filePath = filePath;
    this.cache = null;
    this.writeQueue = Promise.resolve();
  }
  async read() {
    if (!this.cache) {
      this.cache = JSON.parse(await fs.readFile(this.filePath, 'utf-8'));
    }
    return this.cache;
  }
  async write(data) {
    this.cache = data;
    this.writeQueue = this.writeQueue.then(() =>
      fs.writeFile(this.filePath, JSON.stringify(data, null, 2))
    );
    return this.writeQueue;
  }
}
```
