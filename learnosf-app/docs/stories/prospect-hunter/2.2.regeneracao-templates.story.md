## Story
**As a** usuário,
**I want** poder regenerar a mensagem de um lead com instruções diferentes e customizar o template de prompt,
**so that** ajuste o tom ou abordagem quando necessário e personalize a estratégia de comunicação.

## Acceptance Criteria
1. Endpoint `POST /api/leads/:id/regenerate-message` aceita body opcional `{ customPrompt?: string }`
2. Antes de gerar nova mensagem, a mensagem anterior é salva em `message_history` (array de objetos com `content`, `generated_at`, `prompt_used`)
3. `GET /api/config/prompt-template` retorna o template padrão atual de `data/config.json`
4. `PUT /api/config/prompt-template` aceita body `{ template: string, service_description?: string, service_name?: string }` e salva em `data/config.json` com `updated_at` atualizado
5. O template suporta variáveis `{{nome}}`, `{{bio}}`, `{{plataforma}}`, `{{service_description}}`
6. Se `customPrompt` é fornecido no body, ele substitui o template padrão apenas para esta requisição (não salva)
7. Retorna `{ success: true, data: { id, message } }` com a nova mensagem gerada

```json
  "prompt_template": "string com {{variáveis}}",

```

```
POST /api/leads/:id/regenerate-message   body: { customPrompt?: string }
```
