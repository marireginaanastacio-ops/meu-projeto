## Story
**As a** usuário,
**I want** ver métricas do meu pipeline e disparar novas buscas de prospects diretamente do dashboard,
**so that** monitore meus resultados e alimente minha base sem sair da interface.

## Acceptance Criteria
1. `DashboardPage.jsx` exibe cards de métricas: Total de Leads, Novos, Contatados, Convertidos, e Taxa de Conversão (%) com ícones e cores diferenciadas
2. Gráfico de barras ou pizza simples mostrando distribuição de leads por plataforma (Instagram vs LinkedIn)
3. Formulário "Nova Busca" com campos: plataforma (select: Instagram | LinkedIn), palavras-chave (input texto), limite (number input, default 50, max 200)
4. Submit do formulário aciona `POST /api/prospects/search`, exibe barra de progresso/loading e mensagem "Buscando X prospects no [plataforma]..."
5. Ao concluir a busca, exibe resultado: "X leads adicionados, Y duplicatas ignoradas" e lista de leads atualiza automaticamente
6. Hook `useStats()` implementado: busca `GET /api/leads/stats`, retorna métricas formatadas
7. Navegação entre Dashboard e Lista de Leads via tabs ou sidebar (rota `/` = dashboard, `/leads` = lista)

## Business Value
Fecha o produto como sistema completo: o usuário monitora seu pipeline (quantos leads, qual taxa de conversão) e dispara novas buscas sem sair da interface. É o painel de controle que torna o produto autossuficiente.

## Scope
**IN:**
- Cards de métricas: total, novos, contatados, convertidos, taxa de conversão
- Gráfico de distribuição por plataforma (CSS puro, sem biblioteca externa pesada)
- Formulário de nova busca com loading state e feedback de resultado
- Rota `/` para DashboardPage, `/leads` para LeadsPage
- Navegação entre páginas

**OUT (não inclui esta story):**
- Configuração visual de prompt template (apenas leitura/escrita de texto)
- Histórico de buscas anteriores
- Exportação de métricas (CSV, PDF)
- Alertas ou notificações automáticas

- **Requer Story 1.3** — `GET /api/leads/stats` deve existir


### Gráfico de Plataforma sem biblioteca externa
```jsx
// PlatformChart.jsx — barras com CSS
const total = instagram + linkedin;
const instaPct = Math.round((instagram / total) * 100);
const linkedPct = 100 - instaPct;

return (
  <div>
    <div className="flex items-center gap-2 mb-1">
      <span className="w-24">Instagram</span>
      <div className="flex-1 bg-gray-100 rounded h-4">
        <div className="bg-pink-500 h-4 rounded" style={{ width: `${instaPct}%` }} />
      </div>
      <span>{instagram} ({instaPct}%)</span>
    </div>
    {/* LinkedIn similar */}
  </div>
);
```

### Taxa de Conversão
`conversionRate = ((Convertidos / total) * 100).toFixed(1) + '%'`
Exibir como `"8.0%"`. Se total=0, exibir `"—"`.

### Nota das Stories Anteriores
- `api.js` já tem `getStats()` e `searchProspects(params)` (implementados na Story 3.1)
- `useLeads` tem `refetch` — após busca bem-sucedida, se o usuário navegar para `/leads`, a lista estará atualizada
- Para atualização imediata da lista ao buscar: pode usar React Context ou callback via props dependendo da implementação do layout
