## Story
**As a** usuário,
**I want** filtrar meus leads por plataforma e status, e buscar por nome ou bio,
**so that** encontre rapidamente os prospects que preciso sem precisar rolar por centenas de cards.

## Acceptance Criteria
1. Filtro por plataforma: chips/botões `Todos | Instagram | LinkedIn` — atualiza lista ao clicar
2. Filtro por status: chips/botões `Todos | Novo | Contatado | Respondeu | Convertido | Descartado` — atualiza lista ao clicar
3. Campo de busca filtra por nome e bio em tempo real com debounce de 300ms
4. Filtros combinam usando lógica AND (ex: Instagram + Novo mostram apenas leads do Instagram com status Novo)
5. Contador dinâmico exibe "X leads encontrados" atualizado conforme filtros mudam
6. Os filtros são enviados como query params para `GET /api/leads` (server-side filtering)
7. Ao limpar todos os filtros, lista volta ao estado completo
8. URL atualiza com filtros selecionados (permite compartilhar/recarregar com filtros preservados)

## Dev Notes

### Filtros como Query Params (Source: architecture/prospeccao-automatizada-architecture.md#4)
```
GET /api/leads?platform=instagram&status=Novo&search=maria&limit=50&offset=0
```

O backend já suporta estes params desde a Story 1.3. O frontend apenas precisa construir a URL corretamente.

### Debounce Pattern (sem biblioteca externa)
```javascript
// client/src/hooks/useSearch.js
const debounceRef = useRef(null);

const handleSearchChange = (value) => {
  clearTimeout(debounceRef.current);
  debounceRef.current = setTimeout(() => {
    setFilters(prev => ({ ...prev, search: value }));
  }, 300);
};
```

### Acessibilidade dos Chips (NFR WCAG AA)
Usar `<button>` com `aria-pressed="true|false"` para chips de filtro.
Alternativamente, grupo de `<input type="radio">` visualmente estilizados como chips.

### Cores dos Status (Source: Story 3.1 Dev Notes)
Reutilizar as mesmas cores definidas na Story 3.1 para badges de status.

### Nota da Story 3.1
`useLeads` já implementa `refetch(params)`. Esta story apenas precisa conectar
os filtros ao `refetch` através do `useSearch` hook.

### Testing
- Validação manual: testar cada filtro individualmente e combinado
- Verificar que debounce não dispara requisição a cada keystroke
- Verificar URL atualiza corretamente
