## Story
**As a** usuário,
**I want** ver os detalhes completos de um lead, ler a mensagem gerada e atualizar seu status,
**so that** gerencie meu pipeline de prospecção e saiba exatamente o que fazer com cada prospect.

## Acceptance Criteria
1. Clicar em um LeadCard abre painel lateral (drawer) com dados completos do lead: nome, username, bio completa, URL do perfil, foto (se disponível), seguidores, localização, plataforma, data de captura
2. Painel exibe a mensagem gerada (se existir) com botão "Copiar Mensagem" que copia para clipboard e exibe feedback visual "Copiado!" por 2 segundos
3. Se não houver mensagem, botão "Gerar Mensagem" aciona `POST /api/leads/:id/generate-message` com loading state
4. Botão "Regenerar Mensagem" aciona `POST /api/leads/:id/regenerate-message` com loading state
5. Dropdown de status atualiza via `PATCH /api/leads/:id` sem recarregar a página, com feedback visual de sucesso
6. Campo de notas (textarea) permite adicionar/editar notes do lead, salvo via `PATCH /api/leads/:id` com debounce de 1s
7. Botão "Descartar Lead" abre dialog de confirmação antes de chamar `DELETE /api/leads/:id`. Após deletar, fecha o painel e remove o card da lista
8. Painel fecha ao clicar no overlay ou pressionar Esc

## Business Value
Fecha o loop de prospecção: o usuário vê o lead, lê a mensagem gerada, copia para enviar manualmente e registra o status. Sem esta story, o produto não tem utilidade prática no dia a dia.

## Dev Notes

### Endpoints Utilizados (Source: architecture/prospeccao-automatizada-architecture.md#4)
```
PATCH /api/leads/:id              body: { status?, notes? }
DELETE /api/leads/:id
POST /api/leads/:id/generate-message
POST /api/leads/:id/regenerate-message   body: { customPrompt? }
```

### Clipboard API
```javascript
await navigator.clipboard.writeText(text);
// Fallback para browsers antigos:
// document.execCommand('copy')
```

### Status Values (Source: prd/prospeccao-automatizada-PRD.md#FR6)
`Novo | Contatado | Respondeu | Convertido | Descartado`

### shadcn/ui Componentes Necessários
- `Sheet` (drawer lateral) — `npx shadcn@latest add sheet`
- `Select` — `npx shadcn@latest add select`
- `Dialog` — `npx shadcn@latest add dialog` (confirmação de delete)
- `Textarea` — `npx shadcn@latest add textarea`
- `Toast` — `npx shadcn@latest add toast` (feedback de ações)

### Nota das Stories Anteriores
- `api.js` já tem `generateMessage(id)`, `regenerateMessage(id, customPrompt)`, `updateLead(id, data)`, `deleteLead(id)`
- `useLeads` retorna `refetch` — usar `onUpdate`/`onDelete` para atualização local preferível (evita refetch completo)

### Acessibilidade (NFR WCAG AA)
- Sheet/Drawer: deve ter `role="dialog"`, `aria-modal="true"`, focus trap quando aberto
- Confirmação de delete: `role="alertdialog"`
- Botão de copiar: `aria-live="polite"` para anunciar "Copiado!" para screen readers

### Testing
- Validação manual de todos os fluxos
- Verificar que Esc fecha o drawer
- Verificar que delete remove corretamente da lista
